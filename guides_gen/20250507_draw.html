# Terminax `draw` Command Reference

The `draw` command lets you render simple vector shapes directly onto a Terminax window at runtime. You can draw filled shapes, outline (border) shapes, and clear all drawings. This reference covers every option and variation.

## Prerequisites: Active Window

All `draw` commands apply to the **active window**. To set it:

```terminax
$myWin = window(50,50,400,300)
$otherWin = window(500,100,300,200)

// Make $myWin the active drawing target:
$myWin      // a window variable on its own line → now active
draw ...

Any subsequent draw calls will render into the last window variable you placed on its own line. Make sure you haven’t changed active windows unintentionally.

General Syntax

draw <shape>[-border], x, y, width, height[, color][, thickness]
  • <shape>: one of the supported shapes (see below).
  • -border (optional): append “-border” to draw only the outline.
  • x, y: coordinates within the window’s client area.
  • width, height: size of the shape in pixels.
  • color: the drawing color (required for both filled and border shapes).
  • thickness (only with -border): stroke width in pixels.

To clear all drawings:

draw clear

Filled Shapes

By default (no -border suffix), shapes are solid-filled with the given color.

Ellipse / Circle

  • Keyword: ellipse
  • Also valid: circle
  • Syntax:
  draw ellipse, x, y, width, height, color
  // or
  draw circle, x, y, diameter, diameter, color

Rectangle / Square

  • Keyword: rectangle
  • Also valid: rect, square
  • Syntax:
  draw rect, x, y, width, height, color
  // or
  draw square, x, y, side, side, color

Outline (Border) Shapes

To draw only the outline, append -border to the shape name and supply a thickness:

draw rectangle-border, x, y, width, height, color, thickness
draw ellipse-border, x, y, width, height, color, thickness
  • Thickness is in pixels, e.g. 1 for a one-pixel stroke.

Supported outline shapes:

  • ellipse-border, circle-border
  • rectangle-border, rect-border, square-border

Clearing Drawings

Remove all shapes from the active window’s drawing layer:

$myWin   // ensure correct window is active
draw clear

This resets the drawing surface; subsequent draw commands start fresh.

Color Specification

Colors follow the same conventions as the color() style function:

  1. Named colors (built-in):
   red, blue, green, black, white, teal, orange, …
  1. Hex codes (with #):
   "#FF0000", "#00F", "#abcdef"
  1. RGB triples (strings):
   "255,0,0", "0,0,255", "128,128,128"

You may optionally quote names, but plain identifiers work for standard names.

Examples

// --- Setup
$win = window(100,100,600,400)
$win    // active window

// --- Filled shapes
draw ellipse, 50, 50, 200, 100, red
draw rect,    300, 50, 150, 150, "#00FF00"
draw circle,  100, 200,  75,  75, "0,0,255"

// --- Outline shapes
draw rectangle-border, 300, 250, 200, 100, black, 3
draw circle-border,    150, 300, 100, 100, white, 2

// --- Clear all
draw clear

Tips & Best Practices

  • Order matters: later draws appear on top of earlier ones.
  • Play Mode only: draw commands take effect at runtime, not in Edit Mode.
  • Coordinate origin: (0,0) is top-left of the window’s client area.

With these options you can render simple but effective HUD elements, debug overlays, or dynamic indicators in your Terminax scripts. Happy drawing!
